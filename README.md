# Dataset-Helper-v2

**AI 이미지 학습용 데이터셋을 위한 파일명 정리, 태깅, 단일 파일 정리, 이미지 변환, 중복 제거, 분석, 검색·분류, XY표 생성까지 한 번에 처리하는 통합 데스크톱 도구**

PySide6 기반의 모던 UI로 제작되었으며, 라이트(눈부심을 낮춘 부드러운 크림 톤)와 다크(차분한 차콜+블루 톤) 테마를 상단 툴바의 버튼 한 번으로 전환할 수 있습니다. 모든 무거운 작업은 백그라운드 스레드에서 처리되어 작업 중에도 프로그램이 멈추지 않습니다.

---

## ✨ 주요 기능

### 0. 기본 사용법
화면 상단에서 데이터셋이 있는 작업 폴더를 선택하고, 병렬 처리에 사용할 코어 수를 설정합니다. 이 두 값은 대부분의 탭에서 공통으로 사용됩니다.

### 1. 이름 변경
데이터셋 파일들의 이름을 규칙적으로 정리합니다.
- **이미지-텍스트 쌍(Pair) 유지:** 이미지 파일 이름을 변경할 때, 짝이 되는 캡션(.txt) 파일도 자동으로 함께 변경되어 데이터셋의 연결이 끊어지지 않도록 보장합니다.
- **충돌·실패 안전 처리:** 고유 임시 이름을 거쳐 변경하므로 기존 임시 파일명과 충돌하지 않으며, 처리 실패 시 가능한 파일을 원래 이름으로 복구합니다.
- **안전한 실행 취소(Undo):** 실수로 이름을 잘못 변경했더라도, 작업 이력을 통해 즉시 이전 상태로 복구할 수 있습니다. 복구가 일부 실패하면 실행 취소 이력을 보존해 재시도할 수 있습니다.
- **실시간 미리보기:** 변경될 파일명을 미리 리스트로 확인하여 의도한 대로 변경되는지 검증할 수 있습니다.

### 2. 단일 파일 찾기
학습에 불필요하거나 오류를 유발할 수 있는 데이터를 정리합니다.
- **짝 없는 파일(Orphan Files) 감지:** 캡션 텍스트가 없는 이미지나, 이미지가 없는 텍스트 파일을 자동으로 찾아냅니다.
- **격리 및 삭제:** 발견된 파일들을 일괄 삭제하거나, 검토를 위해 별도의 폴더로 이동시킬 수 있습니다.
- **이동 충돌 방지:** 대상 폴더에 같은 이름이 있으면 기존 파일을 덮어쓰지 않고 숫자를 추가해 보존합니다.
- **하위 폴더 포함 검색:** 옵션을 통해 하위 폴더 깊숙이 숨어있는 파일들까지 모두 스캔합니다.

### 3. 태그 처리
Danbooru 스타일 태그 및 자연어 캡션을 효율적으로 편집합니다.
- **태그 정렬 및 표준화:** `1girl`, `solo`와 같은 핵심 태그를 캡션의 맨 앞으로 이동시킵니다.
- **자동 태그 주입:** 캡션에 인원수 정보가 누락된 경우, 설정된 값(성별, 인원수)을 기반으로 태그를 자동 생성하여 추가합니다.
- **인접 태그 수정:** 특정 태그를 기준으로 앞이나 뒤에 있는 태그의 접두사·접미사를 수정합니다.
- **CSV 기반 특수 처리:** 사용자가 제공한 CSV 파일을 기반으로 특정 카테고리의 태그들을 일괄 추가, 치환, 삭제합니다. (언더바·공백 및 대소문자 무관 정밀 매칭 지원)
- **안전한 실행 취소(Undo):** 태그 처리 작업 후에도 [실행 취소] 버튼을 통해 이전 상태로 복구할 수 있습니다.
- **하위 폴더 포함 검색:** 옵션을 통해 하위 폴더에 있는 텍스트 파일까지 한 번에 처리합니다.
- **일괄 편집 기능 (조건부 실행 지원):**
  - **치환:** 특정 태그를 찾아 오타를 수정하거나 다른 태그로 일괄 변경합니다.
  - **삭제:** 학습에 방해되는 불필요한 태그들을 제거합니다. (조건부 삭제 지원)
  - **추가:** 필요한 트리거 워드나 속성 태그를 삽입합니다. (조건부 추가 지원)

### 4. 이미지 변환
다양한 소스의 이미지를 학습에 최적화된 포맷으로 표준화합니다.
- **입력/출력 폴더 설정:** 화면 상단의 공통 작업 폴더와 무관하게, 이 탭에서 별도로 입력 폴더와 출력 폴더를 지정합니다.
- **입력 폴더에 출력:** 체크박스를 활성화하면 출력 폴더를 따로 지정하지 않고 입력 폴더에 그대로 저장합니다.
  - **이름 충돌 처리:** 입력 폴더에 출력 시 파일명이 겹칠 경우 패스 / 덮어쓰기 / 숫자 추가 중 하나로 처리합니다. 파일명 접미사가 켜져 있으면 충돌 자체가 발생하지 않으므로 이 옵션은 자동 비활성화됩니다.
- **파일명 접미사 추가:** 변환된 이미지에 접미사를 붙입니다.
- **포맷 변환 및 리사이징:** JPG, PNG, WEBP, GIF 등 다양한 포맷으로의 일괄 변환과 해상도 조절을 지원합니다. 투명 PNG를 JPG로 바꿀 때는 투명 영역을 흰 배경으로 안전하게 합성합니다.
- **메타데이터 보존:** 변환 과정에서 손실되기 쉬운 EXIF 정보는 물론, PNG Info(AI 생성 파라미터), Stealth PNG Info까지 감지해 대상 파일에 최대한 보존합니다.
- **변환 후 원본 삭제:** 변환 완료 후 원본 파일을 삭제할 수 있습니다. 삭제 전 결과 요약(변환 전·후 개수와 용량)을 팝업과 로그로 확인할 수 있습니다.
- **멀티코어 배치 처리:** 상단의 사용 코어 수에 따라 멀티프로세싱으로 대량의 파일을 빠르게 처리합니다. 진행 중 일시정지·재개·중지를 지원합니다.

### 5. 중복 및 유사 이미지 제거
데이터셋의 품질을 저하시키는 중복 데이터를 정밀하게 걸러냅니다.
- **독립 경로 사용:** 상단의 공통 작업 폴더 대신, 이 탭에서 직접 선택한 별도의 폴더를 독립적으로 검색할 수 있습니다.
- **다중 검색 알고리즘:**
  - **완전 중복(MD5):** 해시가 동일한 파일을 찾아냅니다.
  - **유사 이미지(dHash):** 해상도가 다르거나 약간의 변형이 있어도 시각적으로 유사한 이미지를 검출합니다.
  - **태그 내용 기반 검색:** 짝이 되는 캡션(.txt) 파일의 내용을 분석해 태그 구성이 유사한 이미지들을 찾아냅니다. (Jaccard 유사도)
- **유사도 그룹 검색(범위 검색):** 지정된 범위(예: 0~3) 내의 모든 유사도 그룹을 한 번에 검색해 계층적으로 표시합니다.
- **최적화된 성능:** Union-Find 알고리즘으로 범위 검색 시에도 단일 검색과 대등한 속도를 제공하며, 상단의 사용 코어 수를 반영합니다.
- **직관적인 비교 UI:** 검출된 중복 그룹·쌍을 직접 비교하며 유지할 파일을 선택하고, 나머지는 삭제·이동할 수 있습니다.
- **이미지 비율 비교:** 종횡비가 같은 파일끼리만 비교하도록 설정해 불필요한 비교를 줄여 속도와 정확도를 높일 수 있습니다.
- **텍스트 파일 동반 처리:** 이미지를 삭제·이동할 때 짝이 되는 캡션 파일도 함께 처리하는 옵션을 제공합니다.

### 6. 데이터셋 분석
학습 효율을 극대화하기 위해 데이터셋의 구성과 학습 스텝을 정밀하게 분석합니다.
- **버킷(Bucket) 분석:** 이미지를 학습 시 사용되는 버킷 규격으로 분류해 분포를 보여줍니다. 각 폴더별로 몇 종류의 버킷이 쓰이는지 한눈에 파악할 수 있습니다.
- **학습 환경 설정:** 기준 해상도(Area), 버킷 크기 단위(Step), 최소·최대 해상도를 사용자가 직접 조절할 수 있습니다.
- **정밀 버킷팅 로직:** 단순 반올림이 아닌, 기준 해상도의 면적(Area)을 유지하며 원본 비율에 가장 적합한 버킷을 찾아 할당합니다.
- **버킷 비율 미스매치 감지:** 원본과 배정된 버킷의 종횡비 차이가 큰(30% 초과) 이미지를 자동 감지해 전용 팝업과 CSV로 출력합니다.
- **실시간 계산 갱신:** 설정 변경 후 "분석(계산 갱신)" 버튼만 누르면 폴더를 다시 스캔하지 않고 즉시 재계산됩니다.
- **리핏 최적화 자동화 4종:**
  - **리핏 일괄 설정:** 모든 폴더의 리핏을 한 번에 동일한 값으로 변경합니다.
  - **추천 리핏 설정:** 학습 시간(Step) 균형과 버킷 효율을 동시에 최적화합니다.
  - **최적 리핏 설정:** GPU 자원 낭비율을 최소화하는 데 특화되어 있습니다.
  - **균등 리핏 설정:** 모든 폴더의 데이터량을 평균 이상으로 맞추며 최적 버킷 효율을 찾습니다.
- **결과 CSV 출력:** 분석 요약과 폴더별 상세 내역을 엑셀에서 바로 열 수 있는 CSV(`utf-8-sig`)로 출력합니다.
- **데이터셋 스냅샷:** 데이터셋의 구조·수량·용량 정보를 시점별로 저장하고 비교할 수 있습니다.
  - 전용 창에서 [기본 스냅샷] [비교 스냅샷] [차이점 분석] 3개 탭으로 관리합니다. 차이점 분석 탭은 다시 [신규 추가 폴더] [삭제된 폴더] [변경된 폴더] [이동/재구성] 4개 서브탭으로 나뉘어 표시됩니다.
  - 현재 상태를 이름·메모와 함께 JSON으로 저장하고, 추후 다른 시점과 비교해 추가·삭제·이동된 폴더, 증감량·증감율을 자동 분석합니다.
  - 폴더 구조가 크게 바뀌어도 폴더 이름 기준 퍼지 매칭으로 이동·재구성을 감지합니다.
- **데이터셋 일괄이동:** 분석된 폴더들을 원하는 위치로 한 번에 이동·복사합니다.
  - 동작(이동/복사), 중복 폴더명 처리(건너뛰기/숫자 추가/합치기), 설정 리핏 적용 여부(폴더명 앞에 `{리핏값}_폴더명` 형태로 변경)를 선택할 수 있습니다.
  - 실행 전 대상 폴더명·파일 수·용량을 표로 미리 확인할 수 있고, 실행은 백그라운드에서 진행되어 화면이 멈추지 않습니다.
  - "합치기" 선택 시 같은 이름의 파일이 있으면 번호를 붙여 보존하며, 짝이 되는 캡션(.txt) 파일도 함께 처리됩니다.

### 7. 검색 및 분류
데이터셋 파일을 다양한 조건으로 정밀 검색하고 원하는 방식으로 처리합니다.
- **독립 경로 사용:** 상단 공통 작업 폴더 대신, 이 탭에서 직접 지정한 별도의 폴더를 검색 대상으로 사용할 수 있습니다.
- **4가지 검색 조건 (다중 조합 가능):** 파일명 포함 여부, 용량(KB) 범위, 해상도(px) 범위, 태그(`.txt`, `|`로 다중 입력).
- **유연한 조건 결합:** 각 조건마다 미사용 / AND / OR / NOT 모드를 독립적으로 설정할 수 있습니다.
- **결과 표 및 선택:** 파일명·확장자·폴더·용량·해상도·태그 미리보기가 포함된 표를 제공하며, 열 헤더 클릭으로 정렬, 체크박스로 개별 선택, 전체 선택·해제 버튼을 지원합니다.
- **이미지 및 태그 미리보기:** 표에서 항목을 클릭하면 이미지 썸네일과 태그 파일 내용이 즉시 표시됩니다. 미리보기 이미지를 클릭하면 별도의 독립 뷰어 창이 열려 마우스 휠로 확대·축소(가리키는 지점 중심), 드래그로 이동하며 자세히 볼 수 있습니다.
- **3가지 처리 방식:** 삭제 / 이동 / 복사. 이동·복사 시 파일명 충돌을 자동으로 회피합니다. 처리 완료 후 항목별 성공·실패 상세 로그를 별도 창에서 확인할 수 있습니다.
- **처리 대상 선택:** 이미지+태깅 세트 / 이미지만 / 태깅만 중 선택합니다.
- **짝 없는 파일 발생 경고:** 이미지 또는 태깅 중 하나만 처리할 경우, 남게 될 고아 파일 목록을 미리 알려줍니다.

### 8. XY표 만들기
AI 이미지 모델 병합·테스트 결과를 시각적으로 비교하기 위한 XY 비교표를 생성합니다.
- **두 가지 폴더 입력 방식:** 셀프 선택(폴더별 라벨 직접 입력) 또는 폴더 자동 감지(상위 폴더 지정 시 하위 폴더를 모두 불러옴).
- **빈 칸 채우기 방식:** 격자 우선(행×열 고정, 부족한 칸은 NO IMAGE) 또는 데이터 우선(폴더·이미지 수 기준 자동 결정).
- **이미지 정렬 순서:** 기준(이름 / 생성날짜 / 크기)과 방향(오름차순 / 내림차순)을 독립적으로 선택합니다.
- **이미지 배치 방향:** 폴더를 행 또는 열 중 선택하며, 행↔열 스왑 버튼으로 라벨 격자를 즉시 전치할 수 있습니다.
- **라벨 입력 격자:** 우측 패널에서 행·열 수를 입력해 격자를 생성하고, 각 행·열에 라벨을 직접 입력합니다. "자동 격자 생성" 버튼으로 폴더·이미지 수에 맞춘 최적 행·열을 즉시 계산할 수 있습니다.
- **이미지 셀 크기:** 바짝붙이기(원본 크기 그대로) 또는 최장변 기준 정사각형 공간 확보 중 선택합니다.
- **혼합 해상도 처리:** 이미지 크기가 다를 때 기준(최대 / 최소 / 직접 지정)과 처리 방식(스케일+레터박스 / 중앙 크롭)을 별도로 설정합니다.
- **제목:** 표시 여부와 텍스트, 글자크기 자동/직접 설정을 지원합니다.
- **라벨 글자:** 폰트 크기 모드(자동 / 핏 / 직접)와 가로·세로 정렬을 개별 설정합니다.
- **패딩 / 스케일조절:** 셀 간 여백(px)과 전체 출력 이미지의 축소 비율(%)을 설정합니다.
- **저장 옵션:** PNG / WEBP / JPG 포맷, 무손실·품질 설정, 저장 경로 지정 및 덮어쓰기 확인.
- **미리보기:** 별도 창에서 완성본을 그대로 렌더링해 보여주므로 실제 결과와 정확히 일치합니다. 마우스 휠로 가리키는 지점을 중심으로 확대·축소되며, 드래그로 이동할 수 있습니다(더블클릭 또는 `R` 키로 창 크기에 맞춤). 미리보기 저장과 완성본 저장을 각각 지원합니다.

## 🛠 설치 및 실행 방법

### 요구 사항
- Python 3.10 이상
- Windows 운영체제 (다른 OS에서도 동작하나 일부 경로·폰트 처리는 Windows 기준으로 작성됨)

### 직접 실행 (배포용 exe)
- `Dataset-Helper-v2.exe`를 실행합니다. 별도 설치가 필요 없습니다.
- 실행 시 exe와 같은 위치에 `data/` 폴더가 자동으로 생성되며, 설정·로그·실행취소·스냅샷이 모두 이 폴더 안에 저장됩니다.

### 소스 코드로 실행
```bash
# 1. 저장소를 내려받습니다
git clone <repository-url>
cd Dataset-Helper-v2

# 2. 의존성을 설치합니다
pip install -r requirements.txt

# 3. 프로그램을 실행합니다
python main.py
```

소스 실행과 exe 실행은 동작이 완전히 동일합니다. `data/` 폴더의 위치만 "exe가 있는 폴더" 또는 "main.py가 있는 프로젝트 루트"로 자동 결정됩니다.

### 폴더 구조 한눈에 보기
```
Dataset-Helper-v2/
├── main.py            # 프로그램 진입점
├── requirements.txt
├── core/               # 탭 UI와 분리된 순수 비즈니스 로직
├── image/              # 이미지 변환·메타데이터 전용 로직
├── ui/                 # PySide6 UI 전체
│   ├── tabs/            # 8개 탭
│   └── widgets/         # 여러 탭이 공유하는 재사용 위젯
└── data/               # 실행 중 자동 생성 (설정, 로그, 실행취소, 스냅샷)
```
더 자세한 내부 구조와 모듈별 역할, 수정 시 함께 확인해야 할 파일 목록은 `DEVELOPER_GUIDE.md`를 참고하세요.

### 💡 사용 팁
- **테마 전환:** 상단 툴바 오른쪽의 버튼을 누르면 라이트 ↔ 다크 테마가 즉시 전환됩니다. 마지막으로 선택한 테마는 자동으로 저장되어 다음 실행 시에도 유지됩니다. 처음 실행 시 기본값은 라이트 테마입니다.
- **중복 찾기 범위 검색:** '유사도 그룹 검색'을 사용하면 여러 단계의 유사도를 한 번에 확인할 수 있어 데이터셋에 최적인 필터링 값을 찾기 매우 용이합니다. 시작값이 종료값보다 크더라도 자동으로 순서를 교정합니다.
- **멀티코어 설정:** '이미지 변환'이나 '태그 처리' 시 코어 수를 높이면 속도가 빨라지지만 다른 작업이 느려질 수 있습니다. 원활한 멀티태스킹을 원하면 전체 코어 수보다 2~4개 적게 설정하는 것을 권장합니다.
- **검색 및 분류 - 조건 조합:** AND는 "반드시 포함", OR은 "이 중 하나라도 해당하면 포함", NOT은 "해당하면 제외"로 동작합니다. 예를 들어 "용량 AND(100KB 이상) + 태그 NOT(simple background)"로 설정하면 100KB 이상이면서 단순 배경 태그가 없는 파일만 걸러낼 수 있습니다.
- **검색 및 분류 - 해상도 검색 속도:** 해상도 조건은 이미지 파일을 직접 열어 읽기 때문에 파일 수가 많을수록 느려집니다. 상단의 '사용 코어' 수를 높이면 병렬 처리로 속도를 크게 향상시킬 수 있습니다.
- **XY표 만들기 - 큰 표 만들기:** 라벨 입력 격자는 한 번에 최대 50×50까지 만들 수 있습니다. 그보다 큰 표가 필요하면 여러 번에 나눠 만든 뒤 별도로 합치는 것을 권장합니다.
